Software Commons Consortium

The Software Commons Consortium (SCC) promotes sharing in the software industry.

Definition of the Software Commons

From Wikipedia:

The software commons consists of all computer software which is available at little or no cost and which can be reused with few restrictions. It includes open source software which can be modified with few restrictions. Mini-manifesto

We believe that public software sharing is a good practice, for [reasons]. The established brand related to software sharing, Open Source, is interpreted by many to preclude certain kinds of sharing that we nevertheless wish to promote. SCC recognizes the definitions of the Free and Open Source Software (FOSS) movement, while widening our focus to encompass additional sharing practices within the Software Commons.

We especially encourage companies to share their core software products under licenses that grant extensive freedoms while protecting their commercial interests.

Source Available is a related but highly ambiguous category, encompassing much software that is not part of the Software Commons (for example, code that is readable by customers under NDA but not by the public).

Our long-term goal is to grow the Software Commons into a brand as strong as Open Source. Our ideal future headline is along the lines of, “Microsoft Windows is now part of the Software Commons.”

Types of Software Sharing

SCC recognizes five broad categories of sharing practices in the software industry. All software released under any of these practices is part of the Software Commons.

Public Domain
Fewest restrictions. (CC0, WTFPL)
Permissive
Few restrictions. (BSD, MIT, ISC, Apache)
Single Origin
Restrictions on competition. (FSL, BUSL, ELv2)
Open Core
Restrictions on feature set. (GitLab model)
Copyleft
Restrictions to enforce reciprocal sharing. (GPL, AGPL, LGPL, MPL)
