Software Commons

The software commons consists of all computer software which is available at little or no cost and which can be reused with few restrictions. It includes open source software[.] However the commons also includes software outside of these categories[.]

(Wikipedia)

Pitch

Public software sharing is good, for [reasons]. The established brand related to software sharing, Open Source, is interpreted by many to exclude certain kinds of sharing that are nevertheless also good. Software Commons is a brand with a slightly wider focus. It encompasses all Free and Open Source Software (FOSS), along with additional practices. This website exists to educate the software industry about beneficial sharing practices beyond Open Source, particularly the Sustainable Source model.

The Limits of Open Source

Open Source is great for shared infrastructure, and as the foundation for certain kinds of business models (primarily service-based). However, Open Source licenses are a double-edged sword. They create the possibility for communities to form around software projects, but they do not adequately protect against harmful free-riding, which can lead to a tragedy of the commons.

Many Open Source-aligned founders of software companies want to share the code for their core software products, not just infrastructure. Copyleft is one strategy to combat harmful free-riding, but it is not right for every company. Many founders are more aligned with permissive Open Source. Fortunately, there are good alternatives: Open Core and Sustainable Source.

Comparison with Creative Commons

Software Commons is like Creative Commons for software. Creative Commons is explicitly not for software, but it takes in a wider scope of creative works than Open Source does software. The Creative Commons motto is, “Some rights reserved.” It includes licenses that restrict commercial use, and licenses that restrict modification. A strict definition of Open Source does not allow for such restrictions. Software Commons is a superset of Open Source that acknowledges the validity of some restrictions on sharing in order to curtail harmful free-riding.

That said, Software Commons is also unlike Creative Commons. We are not a formal non-profit, and we do not provide a comprehensive suite of licenses. Established groups already do that. For example, the Open Source Initiative defines licenses for the Open Source subset of Software Commons, and SPDX provides exhaustive license classification.

Software Commons is focused on education about sharing practices beyond Open Source, particularly Sustainable Source.

Five Main Types of Software Sharing

There are five broad categories of sharing practices in the software industry. All software released under any of these practices is part of Software Commons.

Public Domain
Fewest restrictions. (CC0, WTFPL)
Open Source - Permissive
Few restrictions. (BSD, MIT, ISC, Apache)
Open Source - Copyleft
Restrictions to enforce reciprocal sharing. (GPL, AGPL, LGPL, MPL)
Open Core
Restrictions on feature set. (GitLab model)
Sustainable Source
Restrictions on competition. (FSL, BUSL, ELv2)

What Software Commons is Not

Source Available
Source Available is a highly ambiguous category, encompassing some software that is part of Software Commons, and much that is not (for example, code that is readable by customers under NDA but not by the public). We recommend against the use of this term.
Closed Source
Closed-source software is not part of the Software Commons.
